本教程假设你已经按《Docker 部署》或《本地开发》把服务跑起来，前端可访问
`http://localhost:7788`、后端 `http://localhost:8000/docs` 正常。

整体顺序：**配模型 → 建项目/章节 → 拆分镜 → 确认信息 → 生成 → 看任务**。

```text
配置供应商与默认模型
        │
        ▼
  创建项目 → 创建章节
        │
        ▼
  导入剧本 → AI 拆分镜
        │
        ▼
 分镜编辑页（准备）：确认资产/对白 → 镜头变 ready
        │
        ▼
 分镜工作室（生成）：关键帧/参考图 → 生成图片/视频
        │
        ▼
   任务中心：查看进度 / 取消 / 回看结果
```

## 第 0 步：先配模型（必做）

Jellyfish 本身不自带大模型，**不配模型则所有 AI 功能不可用**。进入导航中的
**模型管理**（路由 `/models`）：

1. **新建供应商**：填写名称、`base_url`、`api_key`。
   - 已内置两类适配：OpenAI 兼容协议、火山方舟（Volcengine）。
   - 文本、图片、视频可使用不同的 base URL（供应商支持时可分别覆盖）。
2. **在供应商下新建模型**，按类别区分：
   - `text`：用于剧本理解、拆分镜、实体/对白提取。
   - `image`：用于角色/场景/道具/服装/关键帧的图片生成。
   - `video`：用于镜头视频生成。
3. **设置各类别的默认模型**：系统按「文本/图片/视频」三个默认模型来调度任务。
   没有配默认文本模型时，拆分镜会直接报 “No default model configured”。

> 提示：`api_key` 属敏感信息。若部署时配置了 `FIELD_ENCRYPTION_KEY`，密钥会加密入库；
> 详见《环境变量》中的安全相关说明。

## 第 1 步：创建项目与章节

1. 打开 **项目大厅**（`/projects`，首页默认会跳到这里），新建一个项目。
   - 项目带题材/画面风格维度（如真人都市、动漫科幻等），用于统一生成风格。
2. 进入 **项目工作台**（`/projects/:projectId`），在章节区新建章节。
   - 章节是剧本、分镜、生成的工作单位；一部短剧通常按章节推进。

## 第 2 步：导入剧本，AI 拆分镜

在章节内填入剧本正文，交给 AI 处理。系统会：

- 把剧本拆成一个个**镜头**；
- 从中抽取**角色、场景、道具、服装、对白**作为候选；
- 可选地运行**剧本优化、精简、一致性检查**，以及角色画像、场景细节等定向分析。

这些都是异步任务，提交后可在任务中心看进度（见第 5 步）。

## 第 3 步：分镜编辑页 = 准备

进入某个镜头的 **分镜编辑页**（`/projects/:projectId/chapters/:chapterId/shots/:shotId/edit`）。
这里只做一件事：**把信息确认到位**。

- 确认 AI 抽取的**资产候选**：接受、忽略，或链接到已存在的实体；
- 确认**对白候选**：接受或忽略；
- 修正镜头标题、摘要、景别/机位等基础信息；
- 需要时刷新重新提取候选。

当一个镜头的提取确认全部完成，它的状态从 `pending` 变为 **`ready`**。

> 状态语义要分清三件事，别混用：
> - `shot.status`（`pending` / `ready`）只表示**信息确认状态**；
> - **运行时任务状态**来自任务系统（是否正在生成）；
> - **video-readiness** 单独表示**是否具备视频生成条件**。
> 所以 `ready` 不等于“能立刻出片”。

## 第 4 步：管理共享资产（保证一致性）

打开 **资产管理**（`/assets`）。角色/演员、场景、道具、服装是**跨镜头复用**的共享实体：

- 在镜头里把候选链接到这些共享资产，而不是每个镜头各建一份；
- 给资产维护图片（多视图/多角度），可逐步补齐精度；
- 这样多个镜头引用同一资产，人物身份与画面风格更稳定，减少漂移。

这一步可以和第 3 步穿插进行。

## 第 5 步：分镜工作室 = 生成

镜头 `ready` 后，进入 **分镜工作室**（`/projects/:projectId/chapters/:chapterId/studio`）。
这里负责“生成准备 + 执行生成 + 查看结果”：

1. 管理**关键帧、参考图**；
2. 预览镜头的**视频提示词**；
3. 做 **video-readiness 就绪检查**（单镜头或批量预检）；
4. 发起**图片 / 视频生成**任务，支持单镜头与批量；
5. 生成结果会回写到镜头与媒体系统。

如果在工作室发现镜头还没 `ready`，按引导回到分镜编辑页补确认。

## 第 6 步：任务中心，统一看任务

文本/图片/视频生成都是异步任务。任务中心是**通用任务面板**：

- 查看状态、进度、耗时、成功/失败、结果；
- **取消**正在执行的任务；
- 从任务**跳回**对应的项目 / 章节 / 镜头。

业务细节（提示词调试、图文映射等）留在各自的业务面板里，任务中心只展示通用、轻量信息。

## 第 7 步（可选）：剪辑与导出

镜头视频生成完成后，可进入 **视频剪辑**（`/projects/:projectId/editor`）做时间线编排，
继续后续的合成与导出。

## 常见问题

- **拆分镜报 “No default model configured”**：回到第 0 步，确认已设置默认文本模型。
- **图片/视频生成失败或一直 pending**：检查 Redis（Celery broker）与 S3/RustFS 是否就绪、
  图片/视频默认模型与供应商密钥是否正确；本地开发若缺这些依赖，建议改用 Docker Compose。
- **接口都返回 401**：说明开启了访问鉴权（`API_AUTH_TOKEN`）。前端需带同一令牌，
  Compose 部署会自动透传；自定义部署见《环境变量》。

## 进阶：开启访问鉴权与密钥加密（可选）

默认不启用，按需在 `.env` 配置即可，开启后保持向后兼容：

- `API_AUTH_TOKEN`：保护所有 `/api/v1` 接口，前端自动附带 `Authorization: Bearer`。
- `FIELD_ENCRYPTION_KEY`：供应商 `api_key` / `api_secret` 加密落库；
  backend 与 celery-worker 必须使用相同值。

详细说明见《参考 · 环境变量》。
